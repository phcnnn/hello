# 產品需求文件 (Product Requirements Document - PRD)

## 1. 產品基本資訊
* **專案名稱**：情境式智慧導航系統
* **目標受眾**：
  * 大學生（通勤、夜間活動頻繁）
  * 一般通勤族（步行 / 騎車）
  * 對健康與環境敏感者（關心空氣品質）
  * 夜間活動者（重視安全）

## 2. 背景與動機
目前市面上的導航系統（如 Google Maps）主要以「最短時間」或「最短距離」作為路線推薦依據，忽略了使用者在不同情境下的實際需求與感受。例如：
* **夜間行走安全**：部分路段夜間照明不足，可能產生安全隱患。
* **健康與環境因素**：空氣品質對健康的影響。
* **氣候影響**：炎熱天氣下（如夏季台中）對遮蔭路線的強烈需求。

本系統旨在將「環境因素」與「使用者感受」納入考量，提供貼近真實需求的客製化導航體驗。

## 3. 產品目標
1. 提供除了「最快路線」以外的多樣化導航選擇（如：安全優先、舒適度優先）。
2. 整合並視覺化環境資訊（空氣品質 AQI、日照程度、道路安全/路燈資訊等）。
3. 建立「情境權重模型」以計算並推薦最佳路線。
4. 提升使用者對導航系統的滿意度與信任感。

## 4. 需求分析
### 4.1 核心功能需求 (Functional Requirements)
| 功能編號 | 功能模組 | 功能描述 | 負責人 |
| :--- | :--- | :--- | :--- |
| **F-01** | **路線搜尋** | 使用者輸入起點與終點，系統需提供多種路線選擇。 | 鍾云禎 |
| **F-02** | **情境選擇模式** | 提供不同模式供使用者選擇（如：「安全優先」、「空氣品質優先」、「遮蔭優先」）。 | 陳品涵 |
| **F-03** | **多條路線比較** | 同時顯示不同路線的預估時間、空氣品質狀況及日照程度。 | 陳品涵 |
| **F-04** | **地圖顯示** | 於地圖介面上直觀呈現系統推薦的路線。 | 蕭妤芹 |
| **F-05** | **環境資料整合** | 顯示與路線相關的即時環境資訊（如 AQI、溫度、日照等）。 | 鍾云禎 |

### 4.2 非功能性需求 (Non-functional Requirements)
| 類別 | 需求描述 |
| :--- | :--- |
| **效能** | 路線計算與推薦需在 **3 秒內** 完成，確保操作流暢。 |
| **安全性** | 系統 **不儲存** 使用者個人資料，保障隱私安全。 |
| **易用性** | 操作流程簡明直覺，使用者能在 **3 個步驟內** 完成導航設定與啟動。 |

### 4.3 範圍外 (Out of Scope)
為確保專案能於期限內交付，以下項目**不列入**本次開發範圍：
* 即時交通事故分析與避讓。
* 完整的 AI 深度學習模型（本次專案僅使用簡化演算法及權重模型）。

## 5. 系統架構與技術選型
* **前端介面**：HTML / CSS / JavaScript (純前端，無框架)
* **後端服務**：Python + Flask
* **資料庫**：SQLite (存放使用者的地點設定與歷史環境緩存資料)
* **地圖服務**：Leaflet.js (開源地圖框架，具備離線緩存，開發門檻低)
* **第三方 API**：政府開放資料平台 (介接環境部 API 獲取即時 AQI)
* **協作與版本控制**：GitHub (程式碼同步)、LINE / Discord (團隊溝通)

## 6. 風險評估與應對
| 潛在風險 | 發生可能性 | 影響程度 | 因應策略 |
| :--- | :--- | :--- | :--- |
| **外部地圖服務連接不穩** | 低 | 中 | 使用 Leaflet 具備良好的離線緩存機制，並於斷網時顯示友善的預設提示訊息。 |
| **數據運算過於複雜** | 低 | 中 | 採用簡化版的「權重模型」，先將 AQI 或路燈資訊轉換為 1-5 分的權重，再進行路徑標示計算。 |
| **初學者開發進度延遲** | 中 | 中 | 於開發時程中預留最後一週作為「檢核週」，並優先確保核心功能模組的完成。 |
